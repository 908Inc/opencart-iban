# Opendatabot IBAN Invoice для OpenCart

Модуль оплати для OpenCart: створює рахунок IBAN через Opendatabot і перенаправляє покупця на сторінку оплати. За бажанням — фіксує оплату автоматично через автоклієнт Opendatabot (callback після надходження коштів на рахунок).

**Підтримувані версії:**

- **OpenCart 4.x** (PHP 8.1+)
- **OpenCart 3.x** (PHP 7.4+)
- **OpenCart 2.3.x** (PHP 7.0+)

---

## Встановлення

Є **два шляхи** встановити модуль:

- **A. Через адмін-панель** — завантажити готовий `.ocmod.zip`, OpenCart сам розкладе файли.
- **B. Вручну** — розархівувати архів у папку магазину на сервері (наприклад, через FTP / SSH).

Який шлях обирати — залежить від версії OpenCart і ваших умов (див. нижче). Функціонально результат ідентичний.

### Що потрібно перед встановленням

1. **Архів розширення** відповідної версії:
  - `iban_opencart-4.ocmod.zip` — для OpenCart 4.x
  - `iban_opencart-3.ocmod.zip` — для OpenCart 3.x
  - `iban_opencart-2_3.ocmod.zip` — для OpenCart 2.3.x
   Якщо у вас вихідний код з репозиторію — архів збирається скриптом (див. секцію «Для розробників» нижче).
2. **Ключі API Opendatabot** — `x-client-key` та `x-client-name`. Отримати на [iban.opendatabot.ua](https://iban.opendatabot.ua/create-invoice).
3. **IBAN і код** — ваш український IBAN (UA + 27 цифр) та ІПН / ЄДРПОУ (8 або 10 цифр).

---

### A. Встановлення через адмін-панель

#### OpenCart 4.x

1. Увійдіть у **адмін-панель**.
2. **Розширення → Встановлювач** (Extensions → Installer).
3. Натисніть **Завантажити** і виберіть `iban_opencart-4.ocmod.zip`.
4. **Розширення → Розширення** → у полі **Тип** оберіть **Оплата**.
5. Знайдіть **Opendatabot IBAN Invoice** → **Встановити**.
6. Натисніть **Редагувати**, заповніть поля (див. «Налаштування» нижче) і збережіть.
7. За потреби очистіть кеш: **Система → Налаштування → Сервер → Оновлення теми та кешу**.

#### OpenCart 3.x

1. Увійдіть у **адмін-панель**.
2. **Розширення → Встановлювач** → **Завантажити** → `iban_opencart-3.ocmod.zip`.
3. **Розширення → Модифікації** → натисніть **Оновити** (Refresh) — обов'язково, щоб застосувалися OCMOD-модифікації.
4. **Розширення → Розширення** → тип **Оплата** → знайти **Opendatabot IBAN Invoice** → **Встановити**.
5. **Редагувати** → заповнити поля → **Зберегти**.

#### OpenCart 2.3.x

1. Увійдіть у **адмін-панель**.
2. **Extensions → Extension Installer** → завантажте `iban_opencart-2_3.ocmod.zip`.
3. **Extensions → Modifications** → **Refresh**.
4. **Extensions → Extensions** → тип **Payments** → знайти **Opendatabot IBAN Invoice** → **Install**.
5. **Edit** → заповнити поля → **Save**.

---

### B. Встановлення вручну (розпакування на сервер)

Підходить, коли у вас немає доступу до адмін-інсталятора, або він не працює (наприклад, через права доступу).

#### OpenCart 4.x

Архів `iban_opencart-4.ocmod.zip` містить у корені: `admin/`, `catalog/`, `install.json`.

1. Розархівуйте архів. Отримаєте папки `admin/`, `catalog/` і файл `install.json`.
2. У корені магазину знайдіть папку `extension/`. Створіть усередині неї підпапку, наприклад `extension/opendatabot/opencart_iban/`, і покладіть туди вміст архіву (тобто в підпапці з'являться `admin/`, `catalog/`, `install.json`).
3. Увійдіть у адмін-панель OpenCart.
4. **Розширення → Розширення** → тип **Оплата** → знайти **Opendatabot IBAN Invoice** → **Встановити**.
5. **Редагувати** → налаштування → **Зберегти**.

#### OpenCart 3.x

Архів містить папку `upload/` — це "наклейка" на корінь магазину.

1. Розархівуйте `iban_opencart-3.ocmod.zip`. Отримаєте папку `upload/` зі структурою `admin/...`, `catalog/...`.
2. Скопіюйте **вміст** папки `upload/` у корінь магазину (так, щоб шляхи злилися з існуючими: наприклад, `upload/admin/controller/extension/payment/opencart_iban.php` → `ваш_магазин/admin/controller/extension/payment/opencart_iban.php`).
3. Увійдіть у адмін-панель → **Розширення → Модифікації** → **Оновити**.
4. **Розширення → Розширення** → тип **Оплата** → **Opendatabot IBAN Invoice** → **Встановити** → **Редагувати** → зберегти налаштування.

#### OpenCart 2.3.x

1. Розархівуйте `iban_opencart-2_3.ocmod.zip`. Отримаєте папку `upload/`.
2. Скопіюйте її вміст у корінь магазину (аналогічно OC 3).
3. Увійдіть у адмін-панель → **Extensions → Modifications** → **Refresh**.
4. **Extensions → Extensions** → тип **Payments** → **Opendatabot IBAN Invoice** → **Install** → **Edit** → зберегти.

## Налаштування модуля

Адмін-панель → **Розширення → Оплата → Opendatabot IBAN Invoice → Редагувати**.

**Вкладка «Основні» / «General»**


| Поле                    | Опис                                                                                                                                                     |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **IBAN**                | Український IBAN (UA + 27 цифр, без пробілів). Обов'язково.                                                                                              |
| **РНОКПП / ЄДРПОУ**     | ІПН або код компанії (8 або 10 цифр). Обов'язково.                                                                                                       |
| **x-client-key**        | Ключ клієнта API Opendatabot. Обов'язково.                                                                                                               |
| **x-client-name**       | Ім'я клієнта API (наприклад, `public` або назва застосунку). Обов'язково.                                                                                |
| **Призначення платежу** | Текст призначення для рахунку (для кожної мови). Плейсхолдер `{order_id}` підставляється номером замовлення. Якщо немає — номер дописується автоматично. |
| **Статус замовлення**   | Статус, що ставиться одразу при створенні замовлення (рекомендується **Pending**).                                                                       |
| **Статус**              | Ввімкнути, щоб спосіб оплати показувався в чекауті.                                                                                                      |
| **Порядок сортування**  | Порядок серед інших способів оплати.                                                                                                                     |


**Вкладка «Автоклієнт»** — опційно, лише якщо у вас налаштований Автоклієнт на iban.opendatabot.ua.


| Поле                             | Опис                                                                                                                                                   |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Автоклієнт**                   | Увімкнути, якщо ви хочете автоматичне підтвердження оплати через банк-поллінг. При створенні інвойса покупцю буде показана сторінка очікування оплати. |
| **Статус замовлення (оплачено)** | Статус, який встановиться, коли прийде callback про успішну оплату. Рекомендується **Complete** / «Виконано».                                          |
| **Callback URL**                 | URL, який треба скопіювати і вказати у налаштуваннях Автоклієнта на iban.opendatabot.ua (поле «Webhook URL»).                                          |


Після змін натисніть **Зберегти**.

## Валюта UAH

Спосіб оплати показується **лише коли валюта кошика — UAH**.

Перевірте:

1. **Система → Локалізація → Валюти** — `UAH` існує і ввімкнена.
2. **Система → Налаштування → Локальні** — дефолтна валюта `UAH`.
3. Покупець оформляє замовлення у `UAH` (за замовчуванням або вручну на сайті).

Якщо валюта інша — покупець побачить повідомлення, що цей метод доступний лише для UAH, а не загальну «Немає доступних способів оплати».

---

## Усунення несправностей

### «Немає доступних способів оплати»

1. **Валюта не UAH** — див. вище.
2. **Модуль вимкнено** — у налаштуваннях **Статус** = Ввімкнено.
3. **Порожні обов'язкові поля** — IBAN, РНОКПП/ЄДРПОУ, x-client-key, x-client-name.

---

## Обмеження

- Оплата лише в **UAH**.
- Для створення рахунку потрібні PHP-розширення **curl** і доступ до `https://iban.opendatabot.ua` з вашого сервера.
- Callback автоклієнта має досягати вашого сервера публічним URL (не `localhost`).

## Для розробників

### Структура репозиторію

```
opencart-iban/
├── src/
│   ├── oc4/                        код модуля для OC 4.x
│   │   ├── admin/
│   │   ├── catalog/
│   │   └── install.json
│   ├── oc3/upload/                 код модуля для OC 3.x
│   └── oc2_3/upload/               код модуля для OC 2.3.x
├── dev/
│   ├── oc4/                        Docker-стенд для OC 4.x (порт 8080)
│   ├── oc3/                        Docker-стенд для OC 3.x (порт 8081)
│   └── oc2_3/                      Docker-стенд для OC 2.3.x (порт 8083)
├── scripts/                        скрипти збірки ocmod-архівів
├── iban_icons/
├── *.ocmod.zip                     зібрані архіви (в git не потрапляють)
└── README.md
```

### Збірка ocmod-архівів

```bash
./scripts/build-ocmod-zip-oc4.sh      # → iban_opencart-4.ocmod.zip
./scripts/build-ocmod-zip-oc3.sh      # → iban_opencart-3.ocmod.zip
./scripts/build-ocmod-zip-oc2_3.sh    # → iban_opencart-2_3.ocmod.zip
```

Архіви складаються з вмісту `src/oc*/` (без dev-артефактів) і кладуться у корінь репозиторію.

### Локальний dev-стенд (Docker)

Кожна версія має власний самодостатній Docker-стенд: MySQL 8 + Apache + PHP з потрібними розширеннями, CLI-install OpenCart при першому запуску.

**Порти:**

- OC 4.x — `http://localhost:8080/` (admin: `/admin/`, admin/admin)
- OC 3.x — `http://localhost:8081/` (admin: `/admin/`)
- OC 2.3.x — `http://localhost:8083/` (admin: `/admin/`)

**Запуск (на прикладі OC 4.x)**:

```bash
# 1. Скопіювати .env.example → .env (за бажанням, усі дефолти вже в compose)
cp dev/oc4/.env.example dev/oc4/.env

# 2. Підняти стенд з bind-mount-ом src/ у контейнер
docker compose --env-file dev/oc4/.env \
  -f dev/oc4/docker-compose.yml \
  -f dev/oc4/docker-compose.dev.yml \
  up -d --build

# 3. Відкрити http://localhost:8080/
```

Для OC 3.x і OC 2.3.x — аналогічно, замінивши `oc4` на `oc3` / `oc2_3` і порт.

**Зупинити стенд**:

```bash
docker compose -f dev/oc4/docker-compose.yml down      # зберегти volume (БД, файли)
docker compose -f dev/oc4/docker-compose.yml down -v   # видалити volume (наступний up буде "fresh install")
```

### Логіка dev-стенду

- `docker-compose.yml` — базовий (MySQL + OpenCart).
- `docker-compose.dev.yml` — накидає bind-mount-и коду модуля з `src/oc*/` у контейнер; зміни у файлах одразу видно в магазині без пересборки.
- При першому старті entrypoint:
  - копіює файли OpenCart з образу у volume;
  - чекає базу;
  - виконує CLI-install OpenCart;
  - встановлює права `www-data` на webroot (для OCMOD-інсталятора через адмінку);
  - стартує Apache.

### Тестування встановлення через адмінку

Bind-mount dev-композу перекриває файли модуля у контейнері. Тому, щоб протестувати **встановлення через Extensions → Installer** (а не просто розробити код), підніміть стенд **без** `docker-compose.dev.yml`:

```bash
docker compose -f dev/oc3/docker-compose.yml up -d --build
```

Тоді в контейнері немає файлів модуля, і адмін-інсталятор нормально розпакує `.ocmod.zip`.
